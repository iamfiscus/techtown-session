My problem

I don't understand personal finance.

And I didn't love any of the tools out there — they're either too simple, too complex, or they want to manage my money instead of teaching me.

So I built

Wilson

openaccountant.ai — a dev- and agent-centric harness for thinking about your money the way a smart accountant would.

Which got me thinking: how do you make decisions when you're not the expert?

The real problem

I'm not an expert.

I needed a system — a way to brainstorm, prioritize, and act on complex problems without 20 years of domain knowledge.

The breakthrough

Thinking frameworks

Downloaded github.com/lyndonkl/claude — a library of 95 reusable thinking frameworks, packaged as skills.

Prioritization. Fermi estimation. Decision matrices. Root cause analysis. All the tools a senior operator would reach for, available on tap.

What's a skill?

Just structured markdown.

No code. No API. No plugin architecture.

A markdown file with a description the LLM uses to decide when to load it, and a workflow it follows once loaded.

If your LLM can read, it can run a skill.

---
name: brainstorm-diverge-converge
description: Use when generating
  creative options before narrowing
  to the best choices...
---

# Brainstorm Diverge-Converge

## Workflow
1. Diverge — generate 20-50 ideas
2. Cluster — group into themes
3. Converge — score and select
...

Setup · Claude Desktop

Install once, use everywhere

  1. Open Claude Desktop → Settings → Plugins / Skills
  2. Search thinking-frameworks (or add from lyndonkl/claude)
  3. Click install
  4. Type "brainstorm" in any chat — the skill activates automatically

No restart. No config file. Just markdown loaded on demand.

The same skill in ChatGPT

Skills are portable text.

ChatGPT doesn't have a skill system. Doesn't need one.

  1. Open the SKILL.md file on GitHub
  2. Copy the full contents
  3. Paste into ChatGPT as Custom Instructions or at the start of a chat
  4. Prompt normally — ChatGPT follows the framework

Same framework, any LLM. No vendor lock-in.

Step 4 · The missing piece

Brainstorm output ≠ build prompt.

The brainstorm gives you a ranked feature list.

Horizons needs a build spec: app name, user, features-as-UI, data shape, stack, zero ambiguity.

Paste the raw brainstorm into Horizons and you'll get generic slop. One extra translation prompt fixes it.

The translator prompt

Take the brainstorm output below and produce a single
paste-ready prompt for Hostinger Horizons that builds
a working MVP in one shot.

The output prompt must:
  1. Name the app + one-line purpose
  2. Describe the user (who, why, context)
  3. List each "Ship with MVP" feature as a concrete
     UI requirement — what's on screen, what data
  4. Specify data shape with realistic mock data
     (include a Seed Data button for 200+ rows)
  5. Name the stack: React + Tailwind + shadcn/ui
     + recharts
  6. Pre-resolve every ambiguity — zero questions

Exclude: auth, backend APIs, multi-user, Month-2
features. MVP only.

Structure: "Build a..." → end with a specific CTA.
