# Build Decision Dashboard in Hostinger Horizons

Build a single-page "Support Triage Decision Dashboard" as a clean, professional web application. Use static data — no backend required.

> **Note:** Replace the placeholder data below with your actual brainstorm + Fermi output.

## Layout

### Top Banner
- Problem statement: "Support team handling 1,400 tickets/week | 22 min avg handle time | 6 agents at $65K | Goal: reduce volume and handle time by 30% in 90 days"

### Left Column — Top 3 Ideas
Display as a ranked list (1, 2, 3) with:
- Idea name (bold)
- One-line description
- A colored badge for priority (e.g., green = top pick, yellow = strong runner-up, gray = worth exploring)

### Right Column — Fermi Cards
One card per idea, each showing:
- **Cost to implement** (dollar amount + person-weeks)
- **Time to first impact** (in weeks)
- **Ticket deflection estimate** (percentage + absolute weekly number)
- **Annual savings** (dollar amount)
- A small confidence indicator (Low / Medium / High)

### Bottom Section
- **Recommended next action:** placeholder text
- **Owner:** [TBD]
- **Deadline:** [TBD]

## Design Requirements
- Clean, modern, professional look (think internal dashboard, not marketing site)
- Use a neutral color palette with accent colors for priority badges and key metrics
- Responsive layout that looks good on both desktop and tablet
- Use clear typography hierarchy — the numbers should be the most prominent elements
- No animations needed — clarity over flash
