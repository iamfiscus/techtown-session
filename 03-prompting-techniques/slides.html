How AI conversations work

Every chat has 3 roles:

SYSTEM

Hidden setup. Lovable sets these for you.

USER

What you type. This is where R-C-T-F lives.

ASSISTANT

AI's response. You can follow up.

Stacking layers = packing more useful context into your USER message.

AI has a memory limit

It's called the context window.

  • Each chat holds a finite amount of text in mind
  • Long chats: early messages start slipping away
  • Why R-C-T-F works: packs critical context up front
  • Why iteration works: each new turn gets focused attention

Tip: when output gets weird, start a NEW chat with your V4 prompt.

Markdown in 90 seconds

4 moves AI reads as structure:

# Header           — major section
- bullet           — list of things
**bold**           — emphasis on a key word
```code```          — exact text or code

That's it. Don't overthink it.

prompting-techniques.vercel.app9 / 28

Why it matters

Without structure

build a booking app for my salon
the salon is solo i do hair
i want online booking with a cancel policy

With markdown

# Project
Booking app for a solo salon

# Business
- One stylist, hair only
- Want public online booking
- Need cancel policy

Same content. Different output quality.

Why XML tags?

  • AI models trained on tons of tagged content (HTML, XML)
  • Tags = clear boundaries: "this is the role, that's the task"
  • Claude is specifically tuned for XML structure
  • ChatGPT, Gemini parse it fine too — no downside
  • The CONTENT inside matters more than the tag NAME

You can use <role>, <my_business>, <the_thing> — whatever's clear to you.
