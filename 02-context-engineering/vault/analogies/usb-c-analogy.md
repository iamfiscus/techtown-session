# USB-C Analogy

> "Remember when every phone had a different charger? USB-C fixed that. [[mcp|MCP]] is USB-C for AI."

## The Analogy

### Before USB-C
- iPhone had Lightning
- Samsung had Micro-USB
- Others had Mini-USB
- Every device = different cable
- Travel bag full of chargers

### Before MCP
- Slack integration for ChatGPT ≠ Slack integration for Claude
- Every AI + tool combination = custom code
- N tools x M AI platforms = N*M integrations
- Fragmented, expensive, brittle

### After USB-C / After MCP
- **One standard plug** that works everywhere
- N tools + M platforms = **N+M integrations** (not N*M)
- Build once, connect to everything
- Industry standard, not a proprietary lock-in

## Why This Matters

Just as USB-C made it practical to charge any device with one cable, MCP makes it practical for **any AI to connect to any tool** through one protocol.

This is why MCP has:
- 97M monthly downloads
- Adoption by ChatGPT, Claude, Gemini, Cursor, VS Code
- Governance by the Linux Foundation

## What It Explains

This analogy makes [[mcp]] instantly understandable for non-technical founders. It communicates:
1. **The problem** — too many incompatible connections
2. **The solution** — one universal standard
3. **The implication** — everything just works together
4. **The permanence** — standards don't go away

## Context Engineering Connection

MCP solves the [[dynamic-context]] problem. Before MCP, getting live data into AI required manual copy-pasting or custom integrations. MCP makes [[tool-context]] accessible through a universal standard.
