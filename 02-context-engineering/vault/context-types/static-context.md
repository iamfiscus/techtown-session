# Static Context

> Analogy: **Your company handbook** — it doesn't change day to day.

## Definition

Static context is **information that doesn't change frequently** — documents, brand guidelines, product specs, pricing sheets, style guides. It's the foundation that most people start with (and often stop at).

## Examples

- Product requirements documents
- Brand voice guidelines
- Pricing sheets and feature comparisons
- Technical specifications
- Standard operating procedures
- Company policies
- API documentation

## How Founders Use It

- Upload pitch deck to [[notebooklm]] for research
- Paste brand voice doc into AI before writing marketing copy
- Include product spec when asking AI to build a feature
- Reference pricing sheet when analyzing competitive positioning

## Strengths

- Easy to prepare — you probably already have these docs
- Stable — doesn't go stale quickly
- High signal — purpose-written documents are usually relevant

## Limitations

- **Not enough alone** — Most people ONLY give AI static context
- **Can become stale** — Even "static" docs eventually need updates
- **Volume problem** — Too many docs → [[attention-budget]] depleted

> "Most people only give AI static context. The magic happens when you add [[dynamic-context]] and [[tool-context]]."

## Operations That Work With Static Context

- [[write-operation]] — Create and maintain static docs
- [[select-operation]] — Choose which static docs are relevant NOW
- [[compress-operation]] — Summarize long static docs before injecting

## Tools

- [[notebooklm]] — Upload massive amounts of static context and query across it
- [[obsidian-vector-brain]] — Store static knowledge in a searchable [[second-brain]]

## Compared to Other Types

| Type | Changes? | Example |
|------|----------|---------|
| **Static** | Rarely | Brand guidelines |
| [[dynamic-context]] | Constantly | CRM data, metrics |
| [[episodic-context]] | Accumulates | Conversation history |
| [[tool-context]] | On-demand | API responses |
